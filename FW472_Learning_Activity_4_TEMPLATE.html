Overview and Data

This exercise will use data from the EPA National Lakes Assessment (NLA) from 2017 (the most recently released year). The NLA is a standardized sampling of the physical, chemical, and biological conditions of 1000+ lakes that are meant to be representative of U.S. lakes as a whole. This sampling takes place every few years and then takes a few more to be released (e.g., the 2017 data were only released in 2022).

We are working with four different data files:

2017 NLA Phytoplankton counts and biomass (long data): “nla_2017_phytoplankton_count-data.csv”

2017 NLA Zooplankton counts and biomass (long data): “nla-2017-zooplankton-count-data.csv”

2017 NLA lake site information: “nla_2017_site_information-data.csv”

2017 NLA water chemistry data for a 100 lake subset that has been cleaned (wide format): “NLA_2017_Chemistry_Subset.csv”

You can check out other data and metadata for the NLA alongside the rest of the National Aquatic Resources Survey

In the pre-exercise section, I will go through an ordination example using the phytoplankton data and then your guided exercise will use the zooplankton data.

This line of code will remove those blank values for algal group in the same way we removed NAs before.

phyto_simp <- phyto_simp[!phyto_simp$ALGAL_GROUP == "",]

We can now use the dcast() function to cast our data from long format to wide. We first specify the data we are manipulating, then add the formula for the cast. Here, we want a wide dataframe of site by species or lake UID ~ ALGAL_GROUP. We specify our value variable, which is biovolume and then a function for aggregating the data if there are multiple values such as sum Because ALGAL_GROUP are our broad categories like blue-green algae or diatoms, we have to specify what we want to do with multiple taxa within the group. Taking the sum makes sense here since we want to get the total biovolume of each group by lake.

phyto_site_sp <- dcast(phyto_simp, UID ~ ALGAL_GROUP,  na.rm = TRUE, value.var = "BIOVOLUME", fun.aggregate = sum)

Next, let’s trim our zooplankton data to just the 100 lake UIDs that are in our water chemistry dataset. We can do this using the syntax %in%, which we can use in place of == when filtering data when we want to compare values to a set of values (like all of the UIDs in the water chemistry data as opposed to a single value):

phyto_site_sp <- phyto_site_sp[phyto_site_sp$UID %in% chem$UID, ]

We can now use a new function merge() to combine these two data sets based on a shared column ID. For the NLA data, all lakes are coded by a unique identifier (UID) that can be used to join the different data sets to one another. In order to do this we have to specify what the two merged data sets are being joined by with “by =”, in this case the UID. If you are familiar with dplyr or other R packages, then you can also use one of the many other functions that can be used to join data. Merging in R or other programs is much less error prone and more efficient since you don’t have to sort or organize the data yourself as in Excel.

Here, we are specifying that we want to merge the chem data set and the phyto_site_sp dataset by the UID.

phyto_chem <- merge(chem, phyto_site_sp, by = "UID")

Now, we can finally ordinate! The metaMDS() function below will actually perform both of the steps we discussed in class in one function. First, it will calculate the Bray-Curtis distances (distance = “bray”) among each site based on their phytoplankon biomasses and then it will try to create the best 2D ordination of these distances among lakes. If you replace “bray” with other distances the function allows then it will be based on that metric instead (the other distance metrics are listed under ?vegdist from the same package).

phyto_ord_bray <- metaMDS(phyto_site_sp, distance = "bray")

We can then make a stressplot and calculate the stress of the ordination. The stressplot shows you how the ordination distance between pairs of lakes are related to their actual calculated dissimilarity in phytoplankton communities. You can see that because we are ordinating, we can only visually approximate the true distance relationships among lakes. The stress value is a quantitative measure of how poor this approximation is. A value of 0.2 is in the “interpret carefully” range and a value of 0.3 or greater is considered a poor ordination.

stressplot(phyto_ord_bray)

phyto_ord_bray$stress
## [1] 0.2007366

Regardless of the stress value, we can now plot our ordination. We can see that each lake is represented by a circle and the distance between each pair of lakes is our approximation of how dissimilar their phytoplankton communities are. Here, I also add a measure of the central location of different algal groups on this plot, although this is not generally considered very useful for interpretation.

plot(phyto_ord_bray, main = "Bray-Curtis NLA Phytoplankton Ordination")

We can also place our water chemistry or other environmental factors on this ordination and plot their vectors. Here, we see how each water chemistry variable correlates with our ordination plotting axes and test if they are significant (p-value < 0.05). The differences among the phytoplankton communities in these 100 lakes is signficantly correlated with pH, DOC, silica, and ammonia. The vectors show that the correlations for DOC, silica, and ammonia are all in the same ordination direction and so the individual contribution of each is not separable here (the analysis fits each variable separately).

ef <- envfit(phyto_ord_bray, chem[, -1], permu = 999)
plot(phyto_ord_bray, display = "sites", main = "2017 NLA Phytoplankton Ordination")
plot(ef, p.max=0.05, col = c("orange", "blue", "green", "purple"))

We can also look at how groups of lakes cluster (or not) in their phytoplankton communities by environmental factors. First, let’s create a column called Hypereutrophic for lakes with chlorophyll above 50:

phyto_scores = as.data.frame(scores(phyto_ord_bray)$sites)
phyto_scores$CHLA <- phyto_chem$CHLA
phyto_scores$Hypereutrophic <- phyto_scores$CHLA >= 50

We can now use our extracted ordination scores and this new column to plot any way we like, and specify our graphical parameters ourselves. For example, here we are color coding the lake ordination by whether or not the lake was hypereutrophic.

ggplot(data = phyto_scores, aes(x = NMDS1, y = NMDS2)) + 
     geom_point(data = phyto_scores, aes(colour = Hypereutrophic), size = 3, alpha = 0.5)

A common visualization technique for ordination is to draw hulls around groups. A basic convex hull simply draws connects all of the outer points for a group (like as if you stretched a rubber band around it). Below is a function to make a slightly smoothed hull around our above plot:

ggplot(data = phyto_scores, aes(x = NMDS1, y = NMDS2)) + 
     geom_point(data = phyto_scores, aes(colour = Hypereutrophic), size = 3, alpha = 0.5) +
      geom_mark_hull(data = phyto_scores,aes(fill = Hypereutrophic),concavity = 10)
